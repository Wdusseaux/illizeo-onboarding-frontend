# Matching buddy IA

Choisir le bon buddy pour un nouveau collaborateur est plus difficile qu'il n'y paraît. L'IA Illizeo analyse les profils, les affinités, la charge de travail et l'historique pour proposer les **3 meilleurs buddies** avec un score de compatibilité de 0 à 100.

## Pourquoi un bon buddy fait la différence

Un buddy mal choisi se traduit par :

- Disponibilité insuffisante (le buddy est lui-même surchargé)
- Décalage de profil (junior vs senior, tech vs business)
- Manque d'affinité culturelle (introverti vs extraverti)
- Distance géographique (impossible de prendre un café)

Un bon buddy crée un effet de fidélisation **immédiat et durable**.

## Les critères pris en compte

L'IA analyse une dizaine de critères pondérés :

- **Ancienneté** du buddy (idéalement entre 1 et 5 ans)
- **Site** (même bureau ou télétravail compatible)
- **Charge de travail** (pas en sprint critique)
- **Historique de buddy** (a-t-il déjà été buddy ? avec quel succès ?)
- **Profil personnel** (extraverti / introverti)
- **Domaine d'expertise** (proche du nouveau collaborateur)
- **Equipe** (idéalement adjacente, pas la même équipe directe)
- **NPS récents** (est-il lui-même engagé ?)
- **Disponibilité dans l'agenda** (charge réunions)
- **Préférences** (a-t-il indiqué vouloir être buddy ?)

## Comment utiliser le matching

À l'assignation d'un parcours d'onboarding :

1. Cliquez sur **Choisir un buddy**
2. Sélectionnez **Suggéré par IA**
3. L'IA présente **3 candidats** classés
4. Pour chaque candidat : **score** (0-100), **forces**, **points de vigilance**
5. Vous validez ou choisissez manuellement

Si aucun candidat ne dépasse 70/100, l'IA vous le signale et propose d'élargir les critères.

## Exemple de suggestion

> **Sophie Martin — Score 87/100**
>
> **Forces** :
> - Même site (Lausanne)
> - 3 ans d'ancienneté (sweet spot)
> - Déjà buddy 2 fois avec succès (NPS 9.5)
> - Profil similaire (back-end developer)
> - Disponibilité agenda OK
>
> **Vigilance** :
> - Démarrage d'un projet critique en S+3 (charge élevée prévue)
> - Plutôt introverte (à anticiper si nouveau collab très extraverti)

> 💡 Astuce : le score est une suggestion, pas une vérité. Vous connaissez vos collaborateurs mieux que l'IA. Si vous sentez que ça ne matche pas, choisissez manuellement.

## Le rôle du buddy

Une fois choisi, le buddy reçoit :

- Email d'**explication** du rôle (durée 3 mois, 30 min / semaine)
- **Guide buddy** PDF
- Accès à un **espace dédié** dans Illizeo (checklist, suggestions de sujets)
- Notifications aux étapes clés du parcours du nouveau

## Reconnaître les buddies

Le rôle de buddy est un investissement personnel à reconnaître :

- Mention dans la **revue annuelle**
- **Bonus** symbolique (jour de congé bonus, bon cadeau)
- **Communauté de buddies** (canal Slack dédié, événements)
- **Visibilité** au all-hands

## Buddy de qualité : NPS post-onboarding

À J+30 et J+90, le nouveau collaborateur note son buddy (1 à 10). Ces NPS alimentent l'IA pour les suggestions futures :

- Buddies avec NPS moyen >8 : priorisés
- Buddies avec NPS moyen <6 : déprioritisés
- Buddies sans historique : score neutre

> ⚠️ Important : ne pas mettre la pression sur les buddies à propos de leur NPS. C'est une métrique d'amélioration continue, pas une évaluation de performance.

## Refuser le rôle de buddy

Un collaborateur peut refuser le rôle de buddy ou indiquer une indisponibilité temporaire (deadline projet, retour de congé) via **Mon espace → Préférences**. L'IA respecte ces préférences.

## Quand pas de buddy ?

Pour certains profils, le buddy n'est pas pertinent :

- Cadres dirigeants (C-level)
- Postes à 1 seule personne (pas de pair évident)
- Stagiaires courts (<3 mois)

Dans ces cas, le maître de stage / mentor formel remplace le buddy.

## Quotas

- **Starter** : non disponible
- **Business** : illimité (charge IA modeste)
- **Enterprise** : illimité
